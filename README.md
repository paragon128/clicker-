# clicker-
🖥️ Mogger Simulator 📌 Nazwa projektu Mogger Simulator Przeglądarkowa gra clicker, bo instalowanie czegokolwiek to już za dużo wysiłku dla współczesnego człowieka. 🎮 Założenia projektu Mogger Simulator to przeglądarkowa gra typu clicker (HTML/CSS/JS), w której gracz rozwija swoją postać poprzez klikanie i ulepszanie jej “mog levelu”. Główne mechaniki: kliknięcia generujące punkty (czyli klasyczne „kliknij i czuj się produktywny”) sklep z ulepszeniami zwiększającymi ilość punktów na klik i automatyzację system poziomów postaci (im wyższy level, tym większy „aura boost”) zmiany wizualne postaci wraz z progresją pasywne generowanie punktów (bo ręka też zasługuje na emeryturę) opcjonalny system prestiżu (reset progresu w zamian za bonusy) Technologie: HTML – bo coś musi istnieć CSS – żeby to nie wyglądało jak strona z 2003 roku JavaScript – żeby liczby faktycznie się zmieniały, co jest zaskakująco trudne dla przeglądarki 🗓️ Plan na 2 tygodnie 🎯 Kamień milowy: w pełni grywalna przeglądarkowa wersja clickera Tydzień 1 – „działa, ale nie mów nikomu jak” stworzenie struktury HTML (UI gry) system kliknięć i licznik punktów w JS podstawowy styl CSS (żeby nie bolały oczy) zapis danych w localStorage pierwszy upgrade: +klik power podstawowy layout sklepu Tydzień 2 – „to już prawie gra, niestety” sklep z ulepszeniami (auto-click, multiplikatory) system poziomów i progresji wizualne zmiany postaci (skin/ikony/tier wyglądu) balans ekonomii gry (czyli gaszenie pożaru, który sam stworzyłeś) system prestiżu (reset + bonus) poprawki UI/UX i bugfixy, których nie było, dopóki nie spojrzałeś 🧠 Cel kamienia milowego Gracz ma: wejść na stronę klikać bez sensu widzieć liczby rosnące wystarczająco szybko, żeby zostać i wracać mimo że „to tylko klikacz” Autorzy: 2D_AZ, MG
